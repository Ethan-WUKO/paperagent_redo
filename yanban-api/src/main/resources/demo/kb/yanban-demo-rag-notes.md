# RAG 与评测笔记

本项目的 RAG 流程分为六步：

1. 文档上传：用户上传研究资料，系统记录文件名、类型、大小和处理状态。
2. 文本解析：后端使用解析器提取正文；图片类材料可以接入 OCR。
3. 切块：文档内容按固定窗口切成知识片段，默认 chunk 大小约 500 个字符。
4. 向量化：每个 chunk 调用 embedding 模型生成向量。
5. 索引写入：chunk 文本、文档 ID、用户 ID、可见性和向量写入 Elasticsearch。
6. 检索与重排：用户提问时先召回候选片段，再结合关键词命中、向量分数和 rerank 结果生成回答依据。

建议的 RAG 评测指标：

- Recall@K：衡量正确证据是否出现在前 K 个召回结果中。
- MRR：衡量第一个正确证据出现的位置。
- Faithfulness：检查回答是否被检索到的证据支持。
- Citation Coverage：检查回答中的关键结论是否覆盖引用来源。

当前 demo 文档中，“组会时间”“RAG 流程”和“项目目标”是适合验证知识库检索的样例问题。
