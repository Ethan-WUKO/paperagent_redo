# v1 上线 Checklist

此清单用于判断当前版本是否可以给客户试用。建议把 v1 定位为“私有知识库问答 + 科研学习助手”，Plan Agent 标记为 beta。

## 1. 产品边界

| 检查项 | 门槛 | 当前建议 |
|---|---|---|
| v1 核心卖点明确 | 是 | 私有知识库问答、普通科研学习问答、文档/路线总结 |
| Plan Agent 是否承诺强执行 | 否 | 标记 beta，不作为 SLA 承诺 |
| 论文工作流是否承诺生产可用 | 否 | 当前仅可作为预览能力 |
| 用户能理解失败状态 | 是 | 前端必须显示失败、降级或部分结果 |

## 2. 工程门禁

| 检查项 | 上线要求 |
|---|---|
| `mvn test` | 必须通过 |
| `pnpm build` | 必须通过 |
| 数据库迁移 | 新环境可从空库启动 |
| `.env.example` | 覆盖必要配置且不含真实密钥 |
| 日志 | 不打印 API key、JWT、用户私有全文 |
| 异常处理 | 模型/搜索/向量化失败不导致无提示卡死 |
| Git 卫生 | 不提交 `target/`、`node_modules/`、真实 `.env`、本地大文件 |

## 3. 安全与权限

| 检查项 | 上线要求 |
|---|---|
| JWT 鉴权 | 未登录接口返回 401 |
| 用户隔离 | 用户不能读取其他用户会话、消息、私有知识库 |
| 文件上传 | 限制大小、类型、路径穿越 |
| MCP filesystem | 只允许白名单路径 |
| API key 存储 | 不明文返回前端，生产环境加密密钥固定 |
| WebSocket | token 校验有效，断线后前端可恢复或提示 |

## 4. RAG 质量门槛

| 检查项 | 上线要求 |
|---|---|
| 命中准确性 | P1 RAG 平均分 >= 4.0 |
| 不知道时诚实 | 不编造知识库不存在的信息 |
| 来源可解释 | 回答能说明依据来自哪个文档或检索片段 |
| RAG 开关 | 禁用知识库后不假装已检索 |
| 检索调试 | 客户/管理员能看到检索结果用于排查 |

## 5. Agent 与工具门槛

| 检查项 | 上线要求 |
|---|---|
| 工具循环 | 有硬上限，不无限调用工具 |
| 搜索降级 | 外网搜索不可用时能说明限制 |
| 子 agent 输出 | 搜索结果应被归纳，不直接把原始列表丢给用户 |
| Plan 执行 | beta 可失败，但必须有状态、失败原因、降级说明 |
| 并行任务 | 不影响依赖正确性，不造成上下文串扰 |

## 6. 前端体验

| 检查项 | 上线要求 |
|---|---|
| 登录注册 | 正常可用，错误提示清楚 |
| 聊天页 | 可新建会话、发送消息、查看历史 |
| 知识库页 | 上传、列表、删除、检索调试可用 |
| 设置页 | 可配置 provider/model/key/maxSteps/RAG 默认 |
| Plan 页状态 | RUNNING/COMPLETED/FAILED/DEGRADED/SUPERSEDED 可理解 |
| 移动端 | 核心流程不严重错位 |

## 7. 运维门槛

| 检查项 | 上线要求 |
|---|---|
| 启动文档 | 新机器可按文档启动 |
| 备份方案 | MySQL、MinIO、ES 有备份恢复说明 |
| 健康检查 | `/actuator/health` 可用 |
| 监控指标 | 至少能从日志判断慢请求、模型失败、工具失败 |
| 成本控制 | 工具循环和模型调用有上限 |
| 灰度策略 | 先给小范围客户试用，收集失败样本 |

## 8. Go / No-Go 判定

| 判定 | 条件 |
|---|---|
| Go | P0 全部通过，自动化通过，RAG/聊天 P1 平均 >= 4.0，无安全泄漏 |
| Limited Beta | P0 通过，但 Plan/Paper/MCP 存在已知限制，可小范围试用 |
| No-Go | 任一 P0 失败，或出现数据泄漏、无响应卡死、构建失败 |
